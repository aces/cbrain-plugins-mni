Civet Options

Not all options listed are provided on CBRAIN
To request additional options, leave feedback.


Pipeline options
  • template: Define the template for image processing (0.50, 0.75, 1.00, 1.50, 2.00, 3.00, 4.00, 6.00), default: 1.00.
  • model: Define the model for image-processing (icbm152nl or icbm152lin), default= icbm152nl.
Stage control
  • reset-from: Restart from the specified stage.
CIVET options
  • interp: Interpolation method from native to stereotaxic space (trilinear|tricubic|sinc), default: trilinear.
  • N3-distance: N3 spline distance in mm (suggested values: 200 for 1.5T scan; 50 for 3T scan), default: 200.
  • headheight: Head height in mm for neck cropping, suggested value: 170; default 0=none, for version 1.1.12 or later.
  • lsq6: Use 6-parameter transformation for linear registration, default -lsq9.
  • lsq12: Use 12-parameter transformation for linear registration, default -lsq9.
  • no-surfaces: Don't build surfaces.
  • thickness: Compute cortical thickness and blur (tlink|tlaplace) (kernel size in mm), version <= 1.1.11 -> default: 20, later version: default: 30.
  • resample-surfaces: Resample cortical surfaces.
  • no-resample-surfaces: Opposite of -resample-surfaces, default.
  • area-fwhm:blurring kernel size in mm for resampled surface areas, default: 40, for version 1.1.11 or later.
  • volume-fwhm:blurring kernel size in mm for resampled surface volumes, default: 40, for version 1.1.11 or later.
  • combine-surfaces: Combine left and right cortical surfaces.
  • no-combine-surfaces: Opposite of -combine-surfaces, default.
VBM options
  • VBM: Process VBM files for analysis. default -no-VBM.
  • no-VBM: Don't process VBM files for analysis.
  • VBM-fwhm: Blurring kernel size in mm for volume. default: 8.
  • VBM-symmetry: Run symmetry tools. default -no-VBM-symmetry.
  • no-VBM-symmetry: Don't run symmetry tools.
  • VBM-cerebellum: Keep cerebellum in VBM maps.
  • no-VBM-cerebellum: Mask out cerebellum in VBM maps. default -VBM-cerebellum.
File options
  • prefix: File prefix to be used in naming output files.
  • id-subdir: Indicate that the source directory contains sub-directories for each id.
